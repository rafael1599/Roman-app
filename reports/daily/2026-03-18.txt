Progress Update — March 18, 2026


Summary:

Set up watchdog-pickd as a persistent service on the Bay 2 MacBook, deployed the auto-combine orders feature to production, applied the database migration, and added total weight display to shipping labels.


--- Details below ---


Bay 2 Setup — Persistent Service:

- Cloned the watchdog-pickd repository fresh on the Bay 2 MacBook and configured the environment (venv, dependencies, .env credentials).
- Ran the watcher for the first time, which automatically installed a launchd service (com.antigravity.watchdog-pickd). This means the PDF watcher now starts automatically on login and restarts itself if it ever crashes — no need to keep a terminal open.
- Tested and confirmed the service survives machine restarts. Orders can continue to be sent from Bay 2 without any manual intervention.


Auto-Combine Orders — Deployed:

- Applied the database migration (combine_meta column) to production via supabase db push.
- Pulled the latest watchdog-pickd code on Bay 2 with the combine logic active. From now on, when multiple PDFs come in for the same customer, the system will automatically merge them into a single order.
- Pending: test with real orders — need different order numbers arriving for the same customer to confirm the combine works end-to-end in production.


3D Printing — Location Numbers:

- All numbers (0–9) are now fully printed and ready to go. Next step is to start labeling the warehouse aisles/islands with them.


Shipping Labels — Weight Display:

- Shipping labels now show "WEIGHT: X LBS" after the LOAD field, both in the on-screen preview and the printed PDF.
- The weight is calculated automatically from each SKU's weight_lbs in the metadata, multiplied by the quantity picked. Different SKUs with different weights are handled individually.
- For now, a general default weight was applied across all SKUs. Still pending: update weight_lbs for e-bikes and kids bikes, which have different weights than standard bikes.
